git 명령어
cmd 창 켜기
cd midproject // midproject 공간으로 경로(각자 경로에 맞게) 이동하기
git branch // 현재 branch 확인하기 (각자 branch로 설정되어있으면 됨)
git checkout (psw9659) // 만약 자기 branch에 없다면 각자 자신의 branch로 이동하기
git pull origin develop // develop branch의 파일 내용을 자신의 작업 공간으로 다운받기
///////// 작업후 ///////////////
git add . // 모든 파일 선택
git commit -m "0223 박상원 작업내용" // 커밋하기, 메세지는 날짜, 이름, 작업내용 형식, 그리고 커밋은 자주 해줘도 됨. 만약 조사지 조회, 등록, 수정, 삭제 기능을 오늘 했다면 커밋은 0223 박상원 조사지 등록, 0223 박상원 조사지 조회, 0223 박상원 조사지 수정, 0223 박상원 조사지 삭제 이렇게 각 기능 코드를 작성했을 때 마다 git add . 와 커밋을 해주는게 좋음
git push origin (psw9659) // 각자 개인 branch에 업로드하기
https://github.com/sixsuya/midproject 이 사이트로 들어와서 pull request 하기
pull request 할 때 branch가 develop < (psw9659) 로 잘 설정 되어있는지 꼭 확인하기, 보통 main < (psw9659)로 되어있어서 수정해줘야함
pull request를 하고 git 담당자에게 merge 해달라고 요청하기



cmd 창을 열고 
backend 경로에서 npm install 한번
frontend 경로에서 npm install 한번 하기

백엔드와 프론트엔드에서 각자 개발하면서 npm install "모듈" 명령어를 통해
자신이 개발하는 기능에 필요한 모듈을 다운 받는 경우가 발생 헐 수 있음

실제로 설치 파일은 node_modules에 들어가고 어떤 파일을 설치했는지 영수증 형태로 package.json에 들어감
그런데 git 공유를 할 때 큰 용량을 지원을 안해주기 때문에 
설치파일이 있는 node_modules가 아닌 영수증 형태인 package.json만 공유가 됨
따라서 cmd 창을 열어서 backend와 frontend 각각 경로에 대해서 
npm install을 통해 설치파일을 최신화 해주는 것이 필요
